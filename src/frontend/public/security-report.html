发现并修复的漏洞

已修复 暴力破解锁定返回 500 而非 403
严重程度：中等
原因：SQLite 存储的 datetime 为 naive (无时区信息)，与 Python 的 datetime.now(timezone.utc) (timezone-aware) 比较时抛出 TypeError，导致 500 而非正确的 403。
修复：在比较前添加 .replace(tzinfo=timezone.utc)，确保类型一致。
文件：src/backend/routers/auth_router.py

安全加固建议（生产环境）

以下为非阻塞性建议，当前开发阶段不影响使用，部署生产环境时建议处理。

建议 CORS 配置收紧
当前 allow_origins=["*"]，允许任意域名跨域访问。生产环境应限制为实际部署域名。
建议 关闭 API 文档公开访问
FastAPI 默认的 /docs 和 /redoc 端点可被任何人访问，暴露所有 API 接口结构。生产环境应禁用或加认证。
建议 添加安全响应头
建议添加 X-Content-Type-Options: nosniff、X-Frame-Options: DENY、Content-Security-Policy、Strict-Transport-Security 等安全响应头。
建议 用户名枚举防护
注册时返回 "用户名已存在" 可被利用枚举有效用户名。建议改为模糊提示如 "注册失败，请稍后重试"。
建议 IP 级别速率限制
当前无注册/登录的 IP 级别速率限制，可能被恶意脚本批量注册。建议使用 slowapi 或 Nginx limit_req 添加限流。
