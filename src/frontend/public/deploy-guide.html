2. 开发环境部署

1
克隆或下载项目
# 进入项目目录 cd F:\CodexWorkspace\Project004_考试系统
2
安装后端依赖
cd src\backend pip install -r requirements.txt
3
初始化数据库并写入测试数据
# 首次启动会自动创建数据库和默认管理员 python -m uvicorn main:app --host 0.0.0.0 --port 8003 # 写入建筑检测行业测试数据（可选） python seed_construction.py

4
安装前端依赖并启动
cd src\frontend npm install npm run dev
前端开发服务器：http://localhost:5173 | 后端 API：http://localhost:8003

3. 生产环境部署

方案 A：一键脚本（推荐快速部署）

# 一键构建 + 部署（含 Cloudflare 隧道） powershell -ExecutionPolicy Bypass -File deploy.ps1 # 不启动隧道 powershell -ExecutionPolicy Bypass -File deploy.ps1 -NoTunnel # 跳过前端构建（已有 dist/） powershell -ExecutionPolicy Bypass -File deploy.ps1 -NoBuild
脚本自动完成：构建前端 → 停止旧服务 → 启动后端(含静态文件服务) → 启动 Cloudflare 隧道

方案 B：Nginx 反向代理（推荐正式生产）

1
构建前端
cd src\frontend npm run build # 产出 dist/ 目录
2
启动后端
# 前台运行 cd src\backend uvicorn main:app --host 0.0.0.0 --port 8003 # 后台运行（Linux） nohup uvicorn main:app --host 0.0.0.0 --port 8003 & # 后台运行（Windows） Start-Process python -ArgumentList "-m uvicorn main:app --host 0.0.0.0 --port 8003" -WindowStyle Hidden
3
配置 Nginx
# /etc/nginx/conf.d/exam.conf (Linux) # 或 C:\nginx\conf\exam.conf (Windows) server { listen 80; server_name your-domain.com; root /path/to/src/frontend/dist; index index.html; # SPA 路由 location / { try_files $uri $uri/ /index.html; } # API 代理 location /api { proxy_pass http://127.0.0.1:8003; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; client_max_body_size 200M; } # 上传文件代理 location /uploads { proxy_pass http://127.0.0.1:8003/uploads; } }
4
重载 Nginx
nginx -t && nginx -s reload

5. 外网访问（Cloudflare Tunnel）

适用于没有公网 IP 或端口被封的情况，通过 Cloudflare 免费隧道实现外网访问。

方式一：临时隧道（trycloudflare.com）

# 下载 cloudflared # Windows: https://github.com/cloudflare/cloudflared/releases # 或使用项目自带的 cf2.exe # 启动隧道（需代理访问 Cloudflare API） set HTTPS_PROXY=socks5://127.0.0.1:7890 cf2.exe tunnel --url http://localhost:8003 # 输出类似： https://xxx-xxx.trycloudflare.com

方式二：绑定自定义域名（cloudflare.com）

  1. 注册 Cloudflare 账号
  2. 添加域名（如 yuzhen.xin），修改 DNS 服务器
  3. 安装 cloudflared 并登录：cloudflared tunnel login
  4. 创建命名隧道：cloudflared tunnel create exam-system
  5. 配置 DNS：cloudflared tunnel route dns exam-system yuzhen.xin
  6. 创建配置文件并运行：cloudflared tunnel run exam-system

方式三：Nginx + 公网 IP

如果有公网 IP，配置 Nginx 反向代理后，将域名 A 记录指向该 IP 即可。需确保 80/443 端口未被运营商封禁。

9. 常见问题

Q: 后端启动报错 "address already in use"

端口 8003 被占用，先停止旧进程：Get-Process python* | Stop-Process

Q: 前端 build 失败

确保 Node.js >= 18，删除 node_modules 重新安装：rm -rf node_modules && npm install

Q: 外网无法访问

检查：1) 防火墙是否放行端口 2) 运营商是否封了 80 端口 3) 域名 DNS 是否生效（nslookup yuzhen.xin）

Q: 忘记管理员密码

直接修改数据库：UPDATE users SET hashed_password='新密码哈希' WHERE username='admin'

Q: 如何添加新科目/题库

登录管理员账号 → 题库管理 → 添加科目 → 添加章节 → 添加题目。也支持 Excel 批量导入。
